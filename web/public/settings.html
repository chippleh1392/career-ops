Market & scoring settings

Commerce and general title lists are separate: jobs that look Shopify/commerce use the commerce module (senior-friendly); everything else uses the general module (down-ranks senior titles). See market-scoring/. Also edits search toggles and profile.yml target roles. .bak backups on save; YAML comments may be lost—use git if needed.

Commerce / Shopify title filter

Used when a posting matches the commerce detector (Shopify, Liquid, etc.). Seniority boost adds score for these title tokens.

General (non-commerce) title filter

Used for typical frontend/product roles without a strong Shopify signal. Seniority penalty subtracts score when these words appear in the title (so mid-level listings rank higher).

Target roles (profile.yml)

Strong bonus in market scoring when the job title contains one of these strings (case-insensitive).

Search queries (portal scan)

Toggles enabled on each entry. Used by npm run scan web search discovery—not every market-import source.

Tracked companies

Enable or disable companies pulled in market-import (API / browser boards). Does not remove jobs already in jobs.jsonl until you re-run import.
